# entrySet () 和 keySet () 的区别

entrySet () 和 keySet () 是 Java Map 接口中两个常用的方法，它们之间的主要区别如下：

  1. 返回值类型不同

entrySet () 返回的是一个包含 Map.Entry 对象的 Set 集合。每个 Map.Entry 对象包括一个键和对应的值。

keySet () 返回的是一个包含键的 Set 集合。这意味着返回的集合中只包含键，而不包含对应的值。

  1. 遍历方式不同

使用 entrySet () 方法可以同时获取 Map 中的所有键值对，遍历起来比较直观，而且在需要获取值时可以直接访问 Map.Entry 对象的 getValue () 方法，代码可读性高。

例如：

使用 keySet () 方法只能访问 Map 中的键，如果需要同时获取键和对应的值，则需要通过访问 Map 中的 get () 方法来获取值，相对来说效率比 entrySet () 差一些，因为 get () 方法底层仍然需要遍历整个 Map。

例如：

  1. 性能差异

由于 entrySet () 方法返回的是 Map.Entry 对象，需要同时包含键和值，所以在遍历高负载 Map 时可以提高性能。

而 keySet () 方法仅返回一个键的集合，如果需要访问相关的值，则需要通过调用 get () 方法进行操作，因此效率相对较低。

综上所述，在需要遍历 Map 中的键值对并且想要直接获取键和值时，建议优先使用 entrySet () 方法。如果只需要遍历键或者只需要获取键对应的值，那么就可以使用 keySet () 方法。
